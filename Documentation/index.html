Das Problem

Xcode-Entwicklung kostet zu viel Zeit.

Täglich dieselben manuellen Schritte, verstreute Tools, unübersichtliche Abläufe — und das kostet wertvolle Entwicklungszeit.

🗂
Zu viele Tools

Terminal, Xcode, Instruments, Organizer, Finder — ständige Kontextwechsel unterbrechen den Entwicklungsfluss.

📋
Wiederkehrende Kommandos

Build-Nummern erhöhen, Caches leeren, Crash-Logs symbolicaten — immer wieder dieselben Terminal-Befehle suchen und eintippen.

⚠️
Fehleranfällige Abläufe

Falsche Flags, vergessene Schritte, falsche Reihenfolge — manuelle Prozesse führen zu Fehlern, besonders unter Zeitdruck.

🔍
Schwer auffindbare Infos

Build-Warnungen zählen, Zertifikate prüfen, Framework-Architekturen lesen — jede Information erfordert eigene Kommandos.

🚀
Langsame Release-Vorbereitung

Export, Upload, Notarisierung, Validierung — der Weg in den App Store ist ein mehrschrittiger, fehleranfälliger Prozess.

🧩
Uneinheitliche Team-Abläufe

Jedes Teammitglied hat eigene Skripte und Workflows. Kein Standard, kein einheitliches Vorgehen, keine geteilte Basis.

Die Lösung

Eine Toolbox. Alle Workflows.

Xcode Developer Toolbox bündelt den gesamten Entwicklungs-Lifecycle in einem interaktiven, tastaturgesteuerten Terminal-Tool.

Drei Modi

Einfach. Komplett. Oder ganz individuell.

Das Tool passt sich dem Workflow an — mit einem täglichen Schnellmenü, einem vollständigen Erweiterungsmodus und einem frei konfigurierbaren Benutzermenü.

Simple Mode
Tägliche Schnellaufgaben

16 der meistgenutzten Aktionen auf einen Blick — für den normalen Entwickleralltag optimiert.

  • Cache-Bereinigung (4 Varianten)
  • Build & Run, Quick Reset, App starten
  • Unit Tests ausführen
  • Simulator-Steuerung
  • Xcode öffnen, schließen, neu starten
  • Screenshot, Dark/Light Mode
Extended Mode
Vollständiger Funktionsumfang

22 Themenbereiche mit über 100 Aktionen — für tiefgehende Analysen, Distribution und erweiterte Workflows.

  • Projektanalyse: 38 Read-only-Abfragen
  • Code Quality: SwiftLint, Periphery, Analyzer
  • Git-Analyse: 55+ Abfragen & Reports
  • Binäranalyse, Signing, XCFramework
  • DocC-Dokumentation generieren
  • Metrics mit 5-Punkte-Qualitätsscore
Custom Mode
Persönliches Menü

Bis zu 20 frei belegbare Slots — beliebige Aktionen und Untermenüs aus allen 22 Bereichen nach Wunsch zusammenstellen.

  • 20 individuell konfigurierbare Einträge
  • Aktionen aus allen Themenbereichen wählbar
  • Reihenfolge frei festlegbar
  • Änderungen werden persistent gespeichert
  • Jederzeit über M) umstellbar

Funktionen

22 Themenbereiche.
Jeder Workflow abgedeckt.

Von der ersten Build-Zeile bis zum App-Store-Upload — alle Schritte des iOS/macOS-Entwicklungsprozesses in einer Oberfläche.

Clean
Cache-Verwaltung

Alle relevanten Xcode-Caches mit Größenangabe im Überblick — einzeln oder komplett bereinigen.

  • Derived Data, Module Cache, Simulator
  • SPM Cache, Xcode Internal Caches
  • xcodebuild clean
  • Cache-Inhalte mit Größen anzeigen
Build
Build & Simulator

Build, Run und Reset in einem Schritt — direkt auf dem gewählten Simulator oder Gerät.

  • Build & Run, App starten, Quick Reset
  • Screenshot, Dark/Light Mode toggle
  • Xcode öffnen, schließen, neu starten
  • Simulator stoppen
Testing
Tests & Berichte

Unit- und UI-Tests ausführen, Code Coverage messen und Reports für CI/CD generieren.

  • Unit Tests, UI Tests, Code Coverage
  • Langsame Tests erkennen
  • JUnit XML-Report, Markdown-Report
  • Dateien ohne Tests finden
Analyse
Code Quality

SwiftLint, SwiftFormat, Periphery und der statische Analyzer — alles integriert, mit mehreren Ausgabemodi.

  • SwiftLint (gruppiert, kompakt, nach Datei)
  • SwiftFormat Dry-Run
  • Periphery: ungenutzten Code finden
  • Statischer Analyzer
Projekt
Projektanalyse

38 Read-only-Abfragen für tiefgehende Einblicke in Struktur, Patterns und Qualität des Projekts.

  • LOC, Strukturtypen, Force Unwraps
  • TODO/FIXME/HACK, lange Funktionen
  • Frameworks, SwiftUI-Properties
  • Ungenutzte Assets, fehlende Referenzen
Metrics
Qualitäts-Metriken

5-Punkte-Qualitätsscore pro Datei und für das gesamte Projekt — mit automatischem Risiko-Ranking.

  • Safety, Maintainability, Readability
  • Architecture Score, Testability
  • Top-10-Risikoliste
  • Markdown-Report exportieren
Git
Git-Analyse

55+ Git-Abfragen in 8 Kategorien — von Status & Branches bis zu Statistiken, Reports und eigenem Commit-Verlauf.

  • Hotspot-Dateien, Commit-Frequenz, Stash
  • Blame, Dateihistorie, Suche (Pickaxe/Regex)
  • PR-Summary, Changelog generieren
  • Eigene Commits nach Zeitraum filtern
Geräte
Physische Geräte

Verbundene Apple-Geräte per devicectl verwalten — App installieren, starten, stoppen und Live-Logs lesen.

  • Alle Geräte anzeigen, Gerätediagnose
  • App installieren / starten / beenden
  • Installierte Apps auflisten
  • Live-Log vom Gerät, Diagnose exportieren
Custom
Benutzerdefiniertes Menü

Bis zu 20 Slots frei belegen — eigene Zusammenstellung aller Aktionen aus sämtlichen Themenbereichen.

  • Aktionen & Untermenüs frei kombinierbar
  • Reihenfolge individuell konfigurierbar
  • Persistent gespeichert pro Projekt
  • Umschaltbar via M) im laufenden Betrieb
Debug
Crash & Diagnose

Crash-Reports symbolicaten, dSYM-Infos prüfen und Gerätediagnosen erstellen — direkt aus dem Terminal.

  • Crash-Report symbolicaten
  • Adressen mit atos auflösen
  • dSYM-Integrität prüfen
  • Simulator-Crash-Logs browsen
Binary
Binäranalyse

Einblick in kompilierte App-Binaries: Frameworks, Architekturen, Symbole und versteckte Strings.

  • Linked Frameworks, Architekturen
  • Segment-Größen, Entitlements
  • Exportierte Symbole, Load Commands
  • Hardcodierte Strings suchen
Release
Distribution

IPA exportieren, auf TestFlight hochladen, notarisieren — und mit einem 10-Punkte-Checklist absichern.

  • IPA exportieren, validieren
  • TestFlight / App Store Upload
  • macOS Notarisierung
  • Release-Checkliste (10+ Punkte)
l10n
Lokalisierung

Lokalisierungsstatus prüfen, fehlende Keys finden und Platzhalter-Inkonsistenzen zwischen Sprachen erkennen.

  • Fehlende Übersetzungen finden
  • Ungenutzte Localization-Keys
  • Inkonsistente Platzhalter erkennen
  • Strings exportieren, generieren
Signing
Code Signing

Zertifikate, Profile und Entitlements auf einen Blick — inkl. Ablaufdaten und Integritätsprüfung.

  • Signing-Zertifikate auflisten
  • Provisioning Profiles mit Ablaufdatum
  • App-Signatur verifizieren
  • Mobileprovision entschlüsseln

Wie man arbeitet

In Sekunden bereit. Sofort produktiv.

Das Tool startet im Projekt-Root, erkennt alles automatisch und ist ohne Konfiguration sofort einsatzbereit.

01
Starten

Im Stammverzeichnis des Xcode-Projekts ausführen: ../toolbox/toolbox — das Tool erkennt das Xcode-Projekt automatisch.

02
Schema & Gerät

Mit S Schema, D Gerät wählen — wird projektweise gespeichert.

03
Menü navigieren

Zahl oder Buchstabe eingeben — direkt in den gewünschten Bereich springen, ohne Scrollen.

04
Aktion ausführen

Live-Ausgabe direkt im Terminal. Alle Aktionen sind interaktiv und SIGINT-sicher.

05
Weiterarbeiten

Nach der Aktion zurück ins Menü — Einstellungen bleiben erhalten. Kein neu starten, kein neu konfigurieren.

$ cd MeinXcodeProjekt && ../toolbox/toolbox

Vorteile

Weniger Ablenkung. Mehr Code.

Xcode Developer Toolbox ist kein Replacement für Xcode — es ist die Ebene, die alles darum herum schlanker macht.

⚡️
Zeitersparnis täglich

Wiederkehrende Aufgaben wie Cache leeren, Build starten oder Zertifikate prüfen — statt Minuten dauern sie Sekunden. Kein Command-Suchen, kein Tab-Wechsel.

🛡
Weniger Fehler

Abläufe sind kodiert, nicht auswendig zu lernen. Die Release-Checkliste mit 10+ automatisierten Prüfpunkten verhindert häufige Fehler vor dem App-Store-Upload.

📊
Bessere Codequalität

SwiftLint, Periphery, statischer Analyzer und 59 eigene Code-Checks sind im täglichen Workflow verankert — nicht nur bei CI-Läufen.

🔍
Schnelleres Debugging

Crash-Reports symbolicaten, Adressen auflösen, Simulator-Logs browsen — alle Debug-Workflows in einer Oberfläche statt in drei verschiedenen Tools.

🚀
Schnellere Releases

IPA exportieren, validieren, hochladen, Changelog generieren — der gesamte Release-Prozess ist in der Toolbox abgebildet, ohne Tool-Wechsel.
